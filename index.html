ecozo.us

❮ ❯

Ecozo is a website I developed with the intent of predicting how much power a given solar panel or wind turbine would produce.

The website uses a total of 4 unique models. Two for solar panel power generation, likewise, two for wind turbine power generation. For both of these pairs of models, one is for short-term predictions(1 hour to 5 days) and the other for long-term predictions(1 month to 1 year).

Django was used for the back end of the website. Heroku was used with Docker images to deploy the website to the internet. Scikit-learn, Pandas, Numpy, and Matplotlib were used to develop the predictive models with proper evaluation. Multiple API's were used for training data and real-time information including: OpenWeatherAPI, NREL's PSM3, NREL's Wind toolkit, and WorldClim's historical climate data.

Chappie

❮ ❯

Chappie is an application that can use images or live video to detect a dome in an image and measure its pixel height and diameter which is then compared to the relative measuring device for real-world measurements.

Chappie was developed for the Mate 2023 competition. It was written originally in Python but has been translated to C++ so it can be used more effectivley and quickly in embedded systems.

Used technologies included OpenCV and CMake.

Coral

❮ ❯

Coral Reefs are at the bottom of many critical food chains. With ocean acidification we are seeing these reefs bleach at unprecedented rates.

For this reason I decided to investigate what factors have the highest correlation with observed bleaching and why? In doing so I've also developed a binary ensemble classifier that predicts the mode of the three optimized sub classifiers predictions reaching 87% accuracy on a perfectly balanced dataset.

For the data analysis and model development I used Scikit-Learn, Matplotlib, Pandas, and Numpy. Additionally the data for Coral Bleaching was provided by the Biological & Chemical Oceanography Data Management.

TerminalTavern

❮ ❯

Terminal Tavern was motivated by my fear of networking and multithreading. Prior to this project concepts such as TCP/IP, DDNS, Nat-type, port-forwarding and multithreading all scared me. In this project I developed an easily configurable multithreaded Server. As well as setup a default Server that runs on my linux machine at terminaltavern.ddns.net(I usually just leave the server off) which is connected through port-forwarding.

The client is allowed to create their own 3x3 character with a username and move around in real-time with other players or chat with them.

NCurses was used for managing the terminal screen. Boost::ASIO was used for the networking processes. CMake was used for making Terminal Tavern cross-compatible on linux and macos.
